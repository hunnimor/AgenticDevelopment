## Skill: Feature Delivery

### Goal

Deliver a production-ready feature from requirements to validated implementation.

### Steps

1. Confirm feature scope and acceptance criteria from PRD.
2. Split into backend/frontend/AI/DevOps/QA subtasks.
3. Scaffold files using `create-file`.
4. Implement logic using `generate-code`.
5. Add or update tests.
6. Execute `run-tests`.
7. If issues found, iterate with focused fixes.
8. Summarize what was delivered and what remains.

### Definition of Done

- Functional requirements implemented.
- Tests pass for changed scope.
- No architectural contract breaks.
- Documentation for changed behavior is updated.
